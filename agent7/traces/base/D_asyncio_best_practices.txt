Based on the provided documentation and best practices for Python's `asyncio` library, here is a numbered list of agreed-upon advice for writing efficient asynchronous code:

1.  **Never Block the Event Loop:** Avoid using blocking calls (like `time.sleep()` or heavy synchronous I/O) directly within an async function, as this halts the entire event loop. Use non-blocking alternatives like `await asyncio.sleep()` or offload blocking operations to a thread pool using `loop.run_in_executor()`.
2.  **Use `asyncio.run()` as the Main Entry Point:** Always start your top-level entry point using `asyncio.run()`. This function handles the proper setup and teardown of the event loop and ensures that resources are cleaned up correctly when the program finishes or encounters an exception.
3.  **Leverage Tasks for Concurrency:** Do not await coroutines sequentially if they are independent. Use `asyncio.create_task()` or `asyncio.gather()` to schedule operations to run concurrently, which significantly improves performance.
4.  **Handle Cancellation Gracefully:** Always be prepared for tasks to be cancelled. Catch `asyncio.CancelledError` within your tasks to perform necessary cleanup of resources before re-raising the exception to ensure the cancellation is properly acknowledged.
5.  **Prefer Async Context Managers:** When managing resources (such as network connections or file handles), use `async with` statements. This ensures that resources are acquired and released correctly within the asynchronous lifecycle.
6.  **Enable Debug Mode During Development:** Use `asyncio.run(main(), debug=True)` while developing. This helps identify common issues such as never-awaited coroutines, tasks that block the loop, or unretrieved exceptions, providing detailed tracebacks that simplify debugging.
